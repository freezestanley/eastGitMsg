# GIT BRANCH RULES

> branch 命名: 
--------------------------------------------------

```
  ex: 类型_需求版本_月日
  branch name: feat_moviev1_0708  // 新功能_电影项目第一期需求_分支生成日期
```
  
  类型: 
  *  feat     功能
  *  release  发布分支
  *  fix      bug 修复分支
  *  test     测试分支


> TAG: 
--------------------------------------------------

```
git tag -a tagName -m "my tag" 
```

当有重大版本更新上线完毕后打上tag标记，备注信息标上本次分支包含的需求版本，注意事项
